## 用户需求描述

实现广联达商城**客户下单全流程**的 UI 自动化测试，覆盖以下操作步骤：

1. **客户登录**：单独封装客户专属登录方法（不直接复用现有 LoginPage.login_as_role）
2. **进入产品分类**：点击"全部产品分类"
3. **选择营销专区**：选择"new营销专区12"
4. **等待产品加载**：等待3秒
5. **验证产品出现**：检查"生态"和"非生态"产品是否出现
6. **点击产品**：产品出现后点击进入产品详情页
7. **添加物料**：在产品详情页点击"添加物料"按钮
8. **返回顶部**：页面向上滑动到顶部
9. **进入购物车**：点击购物车图标/按钮
10. **选择商品**：选择第一个商品复选框
11. **确认订单**：点击"订单确认"按钮
12. **继续下单**：点击"继续下单"按钮

## 补充要求

- 元素定位优先使用文字定位（text=）
- 成功或失败均截图，截图文件按时间戳排序
- "添加物料"的具体选择器后续一起研究确认

## 技术方案

### Tech Stack Selection

- **测试框架**：Pytest 7.4.0 + Playwright 1.38.0（与现有项目一致）
- **设计模式**：Page Object Model (POM)，继承现有 BasePage 基类
- **定位策略**：文字定位用 `text=`，优先 data-testid，兼容 Playwright `get_by_text()`
- **截图方案**：在测试用例层面通过 fixture 或 hook 实现成功/失败均截图，文件名格式 `{status}_{timestamp}_{testname}.png`

### Implementation Approach

**整体策略**：采用分层 POM 架构，为下单流程涉及的每个页面创建独立的 Page Object 类，最终在测试用例中组装完整业务流程。

**关键决策**：

1. **CustomerLoginPage 单独封装**：继承 BasePage，新增 `customer_login()` 方法，内部调用已有的 `login()` 逻辑但增加客户角色专属断言（如登录后跳转到客户首页而非通用欢迎页）。

2. **文字定位方案**：使用 `self.page.get_by_text("全部产品分类").click()` 或 `self.locate("text=全部产品分类").click()`，两种方式均基于 Playwright 内置能力，无需额外依赖。

3. **强制等待3秒**：业务要求等待产品加载，使用 `self.page.wait_for_timeout(3000)`（Playwright 提供的合法等待，非 `time.sleep`）。

4. **滚动到顶部**：使用 `self.page.evaluate("window.scrollTo(0, 0)")`，兼容所有浏览器。

5. **截图增强**：扩展 `conftest.py` 中的 `pytest_runtest_makereport` hook，在 `report.when == "call"` 且 `report.passed` 时也执行截图逻辑；截图文件名加入时间戳前缀确保按时间排序。

**性能考虑**：

- 页面对象方法均返回 `self` 或新页面对象，支持链式调用，减少中间变量
- 使用 `wait_for_network_idle()` 替代固定等待，减少不必要的等待时间
- 截图按需触发（成功/失败各一次），不产生冗余 IO

### Implementation Notes

1. **定位器灵活性**：由于用户确认可以用文字定位，但实际系统的 data-testid 可能已存在，页面对象中同时支持两种定位方式，优先使用 data-testid（若前端已添加）。

2. **添加物料选择器待确认**：ProductPage 中的"添加物料"按钮选择器不在本次实现中硬编码，预留方法签名，实际选择器在用户提供页面截图或访问系统后补充。

3. **截图时间戳格式**：使用 `datetime.now().strftime("%Y%m%d_%H%M%S_%f")[:-3]` 精确到毫秒，确保文件排序唯一。

4. **向后兼容**：新增页面对象不影响现有 LoginPage 和 test_login.py，所有修改为纯增量。

### Architecture Design

```
tests/test_customer_order.py
    ↓ 使用
pages/customer_login_page.py  →  pages/home_page.py  →  pages/product_page.py  →  pages/cart_page.py  →  pages/order_page.py
```

`customer_session fixture` → `CustomerLoginPage.customer_login()` → `HomePage` → `ProductPage` → `CartPage` → `OrderPage` → 断言订单提交成功

## Agent Extensions

### SubAgent

- **code-explorer**
- Purpose: 在实现过程中需要进一步探索项目代码，确认现有方法和命名规范
- Expected outcome: 确保新增代码与现有项目风格一致，正确复用 BasePage 方法
